# Nano Letters 投稿前チェックリスト

## 0. 論文種別・全体方針
- [ ] ジャーナルは **Nano Letters（ACS）** である
- [ ] 論文種別は **Letter（Communication）** である
- [ ] 「速報性・高い新規性」がカバーレターで説明できる

---

## 1. 文字数・構成制限（重要）

### 本文（Main Manuscript）
- [ ] 本文の総語数が **3000 words 以下**（安全側）
- [ ] Abstract が **150 words 以下**
- [ ] 図（Scheme 含む）の数が **5 枚以下**（安全側）
- [ ] Table は必要最小限（原則は Figure 中心）
- [ ] セクション見出しを多用せず、**流れる文章構成**になっている
- [ ] Introduction / Results / Discussion / Conclusion を分けすぎていない

---

## 2. 図・表（Figures & Tables）
- [ ] すべての Figure が **本文中に埋め込まれている**
- [ ] 各 Figure が本文中で必ず言及されている
- [ ] Figure キャプションが **自己完結的**（SIを見なくても理解可能）
- [ ] 軸ラベル・単位・スケールバーが明記されている
- [ ] 解像度が十分（印刷耐性あり）
- [ ] カラーバーの意味が明確
- [ ] Figure 番号の不整合がない

---

## 3. Supporting Information（SI）
- [ ] SI を **本文とは別ファイル**として用意している
- [ ] SI ファイル名が分かりやすい（例：`Supporting_Information.pdf`）
- [ ] 本文中に以下の記述がある  
  - [ ] `Supporting Information: Experimental details, additional figures, ... (PDF)`
- [ ] 本文の主張が **SI に依存していない**
- [ ] SI の Figure / Table 番号が本文と混ざっていない（Figure S1 など）

---

## 4. TOC（Table of Contents）Graphic
- [ ] TOC Graphic を用意している
- [ ] サイズが **3.25 in × 1.75 in 以内**
- [ ] 文字が小さすぎない
- [ ] 1 枚で論文の「核」が分かる構成
- [ ] 装飾過多になっていない

---

## 5. 参考文献（References）
- [ ] 参考文献情報が **完全**（著者・タイトル・誌名・年・巻・ページ）
- [ ] 最新の関連論文が適切に引用されている
- [ ] 自己引用が過剰でない
- [ ] 引用番号のズレがない

---

## 6. カバーレター（Cover Letter）
- [ ] 原稿タイトルを明記している
- [ ] Corresponding author の連絡先を明記
- [ ] 以下を明確に説明している
  - [ ] 何が新しいか（Novelty）
  - [ ] なぜ重要か（Signi
